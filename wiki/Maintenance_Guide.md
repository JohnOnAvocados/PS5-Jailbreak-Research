# Wiki Maintenance Guide

## How the Wiki Works

This wiki has two content sources that coexist:

1. **Hand-curated pages** (this file, Home.md, _Sidebar.md, Topics.md, and all .md pages in `wiki/`)
   - Written and maintained by researchers
   - Pipeline never overwrites them
2. **Auto-generated pages** (`wiki/_Pipeline_Home.md`, `wiki/sections/*.md`)
   - Generated by `python tools/controller.py wiki`
   - `_Pipeline_Home.md` contains graph stats and section links
   - `sections/*.md` contain node listings per system layer

## Running the Pipeline

```bash
python tools/controller.py full_run
```

This runs all steps: ingest → normalize → extract → link → graph → obsidian → wiki → reports

**Partial runs:**
```bash
python tools/controller.py wiki      # Regenerate wiki sections only
python tools/controller.py graph     # Regenerate graph only
python tools/controller.py obsidian  # Regenerate Obsidian vault only
python tools/controller.py reports   # Regenerate pipeline report only
```

### Pipeline Steps
| Step | Input | Output | Description |
|------|-------|--------|-------------|
| ingest | `inbox/*.md` | `intermediate/raw.json` | Read source files |
| normalize | `raw.json` | `intermediate/clean.json` | Deduplicate, clean metadata |
| extract | `clean.json` | `intermediate/structured.json` | Extract wikilinks, concepts |
| link | `structured.json` | Links in memory | Resolve wikilink references |
| graph | Links | `graph/graph.json`, `graph/nodes.json`, `graph/edges.json`, `graph/summary.json` | Build knowledge graph |
| obsidian | Graph | `obsidian/vault/` | Generate Obsidian vault nodes |
| wiki | Graph | `wiki/_Pipeline_Home.md`, `wiki/sections/*.md` | Generate wiki sections |
| reports | Graph | `reports/final_system_summary.md` | Generate summary report |

## Adding New Content

### Add a Source File (Ingested by Pipeline)
Place `.md` files in `inbox/` with frontmatter:
```markdown
# Title
## Source URL
https://example.com
## System Layer
kernel  # or: hardware, firmware, hypervisor, userland, security_model, boot_chain, memory_protection, cpu_architecture, system_overview
## Summary
Brief description
## Key Concepts
- Key point 1
- Key point 2
```

Then run `python tools/controller.py full_run` to integrate.

### Add a Hand-Curated Wiki Page
Create `.md` files directly in `wiki/`. Use `[[wikilinks]]` to cross-reference:
```markdown
# My Topic
## Overview
Related to [[Related_Topic]] and documented in `research/path/to/file.md`
```

The pipeline will NOT overwrite files in `wiki/`.

### Add an Obsidian Concept Note
Create `.md` files in `obsidian/concepts/` with:
```markdown
# Concept Name
## Concept Summary
Brief description.

## Role in System
How this fits in the architecture.

## Connections
- [[other_concept]]
- [[another_concept]]

## Graph Reference
research/path/to/source_file.md
```

## Adding New Source Files to Inbox

1. Fetch or write the source markdown file
2. Place it in `inbox/` with proper frontmatter (Source URL, System Layer, Summary, Key Concepts)
3. Run `python tools/controller.py full_run`
4. Verify new nodes appear in `graph/summary.json`

## Keeping Content Synchronized

The knowledge graph connects:
- **Wiki pages** (`wiki/`) — human-readable reference
- **Research files** (`research/`) — deep technical analysis
- **Obsidian vault** (`obsidian/vault/`) — graph-visualizable notes
- **Obsidian concepts** (`obsidian/concepts/`) — high-level concept summaries
- **Pipeline outputs** (`graph/`, `intermediate/`, `reports/`) — auto-generated

When adding or updating content in any one system, update the others:
1. Write the research file in `research/`
2. Create an Obsidian concept note in `obsidian/concepts/`
3. Add a wiki page in `wiki/` summarizing the key findings
4. Update `wiki/Topics.md` cross-reference if adding major topics
5. Update `wiki/_Sidebar.md` if adding new navigation items
6. Run the pipeline to regenerate auto content
