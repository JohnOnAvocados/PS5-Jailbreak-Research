# SDN-based In-network Honeypot

## Source URL

## System Layer (choose one)

## Summary
Research paper by Hui Lin proposing an SDN-based in-network honeypot that disrupts and misleads IoT network attackers at early stages. Uses Software-Defined Networking to reroute traffic from suspicious nodes and spoof network responses without dedicated hardware.

## Key Concepts
- Software-Defined Networking (SDN) based honeypot deployment
- In-network traffic rerouting from suspicious nodes
- No dedicated hardware requirement
- IoT network attack disruption and misdirection
- OpenFlow flow table manipulation
- Minimal performance impact on IoT networks

## Security Relevance
Demonstrates network-level deception and active defense using SDN programmability. Relevant to network security layers within a trusted system model for proactive threat disruption in IoT environments.
