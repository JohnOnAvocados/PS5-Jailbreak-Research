## Summary
Common Vulnerabilities and Exposures (CVE) is a publicly available dictionary of standardized identifiers for known vulnerabilities and exposures. Maintained by the MITRE Corporation and sponsored by the US Department of Homeland Security (DHS) Cybersecurity and Infrastructure Security Agency (CISA). Each CVE entry has a unique CVE-ID (CVE-YYYY-NNNNN format), a description, and references. The program includes over 200 CVE Numbering Authorities (CNAs) worldwide who assign identifiers.

## Key Concepts
- Standardized CVE-ID identifier format
- Publicly available vulnerability dictionary
- Over 200 CVE Numbering Authorities (CNAs) globally
- Managed by MITRE, sponsored by DHS/CISA
- Machine-readable data feeds and APIs
- Vendor-neutral vulnerability identification
- References to advisories and patch information
- Integration with NVD for enrichment (CVSS scores, CWE)

## Security Relevance
Provides the authoritative standardized identifier system for all known vulnerabilities that threat models must account for. Critical for tracking software/hardware vulnerabilities by CVE-ID, cross-referencing with exploit availability, and determining patch/update requirements for trusted systems.
