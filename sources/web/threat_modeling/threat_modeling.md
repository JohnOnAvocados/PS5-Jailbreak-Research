## Summary
OWASP defines threat modeling as a structured process to identify, communicate, and understand threats and mitigations for protecting valuable assets. It covers software, applications, systems, networks, IoT devices, and business processes. The page describes a four-question framework (scope, what can go wrong, countermeasures, validation) and references structured methods including STRIDE, Kill Chains, and Attack Trees.

## Key Concepts
- Four Question Framework: scope, what can go wrong, countermeasures, validation
- STRIDE, Kill Chains, Attack Trees as structured analysis methods
- Threat model includes: subject description, assumptions, potential threats, mitigations, validation
- Continuous application across software development lifecycle
- Assurance argument for explaining and defending security
- Threat Modeling Manifesto values and principles (2020)
- Prioritized list of security improvements as output

## Security Relevance
Defines the core methodology for systematic security analysis of any trusted system. The four-question framework directly applies to console/hardware threat modeling by structuring how to enumerate attack surfaces, trust boundaries, and mitigation requirements.
