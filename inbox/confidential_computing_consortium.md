## Summary
Linux Foundation project focused on securing data in use through confidential computing technologies. Members include AMD, Google, Intel, Microsoft, Meta, NVIDIA, and others. Produces white papers, common terminology definitions, compliance frameworks, and open-source projects for trusted execution environments (TEEs) and attestation.

## Key Concepts
- Confidential computing encrypts data in use (during processing), complementing encryption at rest and in transit
- Trusted Execution Environments (TEEs) isolate code and data from host OS and cloud provider
- Attestation mechanisms verify TEE integrity to remote parties
- Open-source projects include Verdict, COCONUT-SVSM, and trusted firmware
- Common Terminology for Confidential Computing standardizes industry definitions
- Governance, Risk, and Compliance (GRC) working group for regulatory frameworks

## Security Relevance
Defines the industry architecture for protecting data during computation, critical for trusted cloud models. TEE-based confidential computing is a key mechanism for reducing trust requirements in multi-tenant systems by providing hardware-enforced isolation from the platform owner.
